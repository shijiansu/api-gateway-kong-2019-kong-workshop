Lab Management 1 (Workspaces & RBAC)

Until now we have been making all configurations in the default workspace and the admin API was not secured at all.

  1. Create two new workspaces
  2. (opt) Create a new portal in one of the workspaces
  3. Create a super-admin user

Enabling RBAC in a Docker based deployment

To enable RBAC you have to change a setting in the Kong configuration file (default location /etc/kong/kong.conf) and reload Kong. In a Docker based setup (like the Lab Enviroment) this can also be achieved by changing some environment variables in the container:

  1. docker ps
  2. docker exec -ti kong-ent /bin/sh
  3. KONG_ENFORCE_RBAC=on kong reload
  4. exit
  5. Try to access admin API on port 8001
  6. Get access to admin api using the token of the super-user

See also: https://docs.konghq.com/enterprise/0.36-x/installation/docker/#enable-rbac
