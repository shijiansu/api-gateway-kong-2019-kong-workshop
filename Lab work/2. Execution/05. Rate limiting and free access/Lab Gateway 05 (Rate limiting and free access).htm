Lab Gateway 05 (Rate limiting and free access)

In the moment you add key-auth to a route it will be only accessible if you provide an API key. What about having the API open for initial tests with a very strict rate limiting to make developer onboarding even easier?

  1. Use the previously created route with key-auth and rate-limiting but change it in a way, that without a valid apikey you still can get access to the service.

Hint: Use anonymous consumer to open the api again for non / invalid api keys: https://docs.konghq.com/1.2.x/auth/#anonymous-access and https://docs.konghq.com/1.2.x/auth/#multiple-authentication in the documentation.
